# 机会发现雷达 v2

> "选轻资产低成本,可重试,输了不痛不痒还能涨经验。" —— 原作者 #1

## 这是什么

一个**判断框架**,不是一个内容生成器。你问"有什么副业可做",它不直接告诉你答案——而是给你一套**4 原则 + 6 维打分卡**,让你自己**评估**任何想到的机会。

**核心约束**:

1. **不做**: 股票/币种/具体项目买卖建议;代码/合同/合同书生成;灰产/赌博/色情/盗版内容。
2. **做**: 把用户提到的任何机会,套进 4 原则 + 6 维打分卡,给出**可比较的候选清单**。
3. **判断**: 个别边缘机会(跨境/海外注册/AI 应用)可以提,但必须**明确标注"需自评合规"**。

## 触发场景

- "我手里有 X 万,想搞副业,做什么好?"
- "学什么技能能变现?"
- "有什么跨市场套利的机会?"
- "如何用信息差赚钱?"
- "我能不能做 XX 这个生意?"
- "现在 2026 还有什么风口?"
- "换跑道,我想从 A 转到 B,你看行不行?"

**不触发**: "这个股票能不能买?" / "帮我写代码" / "翻译一下" / "写个商业计划书"。

## 4 原则(主轴)

| # | 原则 | 一句话 | 原作者原文出处 |
|---|---|---|---|
| P1 | **轻资产·低成本·可重试** | 单笔 ≤ 总资产 1%,输了不痛还能涨经验 | #1 #213 #135 |
| P2 | **信息差 = 钱** | 找"我知道别人不知道"的差,做撮合者 | #1 #3 #5 |
| P3 | **跨市场 + 跨境** | 同样能力搬到竞争更弱的市场 = 5x~10x | #1 #213 |
| P4 | **反消费主义** | 花钱不超收入 1/10,赚的滚去再生产 | #124 #105 |

完整原则 + 金句见 [`references/01-principles.md`](references/01-principles.md)。

## 6 维打分卡(用于评估每个候选)

| # | 维度 | 0 分(差) | 10 分(好) |
|---|---|---|---|
| D1 | 试错成本 | 需要贷款梭哈 | ≤ 1% 总资产,输了不痛 |
| D2 | 经验可回收 | 失败 = 钱打水漂 | 失败也涨经验/人脉/技能 |
| D3 | 中间商/撮合度 | 重资产开店 | 纯信息中介/撮合 |
| D4 | 跨市场/跨境加成 | 完全国内封闭 | 完全靠跨境信息差 |
| D5 | 新兴市场 | 房地产/传统餐饮 | AI/独立开发者/新能源 |
| D6 | 可 IP 化 + 全球化 | 一次性卖货 | 个人品牌 + 资产全球化 |

**总分 ≥ 42/60 才推荐**,否则给"为什么不够格"的诊断。

## 3 类应用模式(每条原则的落地变体)

详见 [`references/03-application-patterns.md`](references/03-application-patterns.md)。

- **A1 技能可复用**: 你会的技能(写作/设计/编程/翻译)搬到 X 市场,看哪里 5x 收益
- **A2 跨地区价差**: 同一商品/服务,A 地 100、B 地 800,做搬运工
- **A3 AI 杠杆**: 用 GPT/Claude 把自己的边际时间成本压到接近 0

## 错误模式(警示信号)

详见 [`references/04-red-flags.md`](references/04-red-flags.md)。命中任何一条直接 PASS:

1. 需要贷款梭哈 / 重资产开店
2. 来源不可验证的"内部消息"
3. 一次只能赚一次的零和游戏
4. 需要先付"会员费/培训费/加盟费"
5. 灰色/明确违法(毒品/色情/赌博/盗版本身)
6. 不在新兴市场/已经是红海
7. 没有信息差,只有体力差

## 流程(用户问 → 输出)

### Step 1: 识别用户约束(无信息时用默认画像)

向用户问/推断:
- **本金量级**(< 10 万 / 10-100 万 / 100 万+)
- **技能背景**(写作/设计/编程/翻译/销售/其他)
- **风险偏好**(必须 0 风险 / 可承受波动 / 能扛 -50%)
- **时间预算**(每天 2h 副业 / 全职 / 周末党)
- **地理限制**(人在国内 / 肉身已翻 / 计划出海)

如果用户没说全,给**默认画像**(技能中性,< 10 万本金,副业形式,0 风险承受,国内),并明确写出"假设画像是 XX,如果你不是请告诉我"。

### Step 2: 套框架(必须显式走一遍 4 原则)

不直接答用户。先**过一遍 4 原则**,问自己:
- 这机会能不能用 P1 试错成本公式算?
- 它依赖什么信息差(P2)?
- 它能不能跨市场(P3)?
- 它会不会让人更想消费(P4)?

如果 4 条原则都不沾边,直接告诉用户"这机会不符合本框架,继续聊也浪费时间"。

### Step 3: 输出 3-5 个候选(每个含 6 维打分)

每个候选包含:
1. **名字**(一句话描述)
2. **本金门槛 / 试错成本**(具体数字)
3. **预期年化 / 月化区间**(基于同类案例的实测)
4. **6 维打分**(D1~D6 + 总分)
5. **第一周做什么**(≤ 5 步,可执行)
6. **反模式检查**(7 条全部勾一遍)
7. **合规标注**(边缘机会明确标"需自评合规")
8. **风险与黑天鹅**

### Step 4: 排序 + 提醒

按总分排序。提醒:
- 同时跑 2-3 个候选,**不 all-in**
- 每个项目预算上限 = 总资产 1%
- 任何项目连续 2 个月无气色就砍掉

## 边界与失败模式

1. **不推荐具体股票/币种** —— 本 skill 只做机会评估框架,不做交易信号
2. **不主动写代码** —— 即使候选是"做个 SaaS",代码也是用户自己的事
3. **如果用户问的是"该不该买 X 股票"** —— 直接拒,告诉用户这超出 skill 范围
4. **如果用户问"X 灰色生意能做吗"** —— 答"明确违法 → 不做;灰色边缘 → 给"需自评合规"标注 + 风险清单",不主动展开
5. **如果用户预算 ≥ 1000 万** —— 提示进入"机构资产配置"范畴,本 skill 不适用,建议找专业顾问
