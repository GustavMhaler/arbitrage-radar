# 03 — 3 类应用模式(每条原则的落地变体)

> 4 原则是判断主轴,但落到具体执行时有 3 类反复出现的应用模式。
> 评估一个候选商机时,先看它属于哪一类,再用对应模式细化。

---

## A1. 技能可复用(原 #1 + #213 强调 "找到擅长领域,打磨成顶尖专家")

### 适用场景

你已经会一项技能(写作/设计/编程/翻译/剪辑/教培),想把这技能换种方式变现。

### 三个具体操作

#### A1.1 把国内技能搬到英文市场

- **前提**: 你的技能在中国是普通水平,但放到 Upwork / Fiverr / Twitter 海外圈,可能是"专家级"溢价。
- **例子**:
  - 中国电商详情页设计 → Upwork 给海外小品牌做 Shopify 模板
  - 中文短视频剪辑 → 给海外 YouTuber / TikToker 剪英文视频
  - 中文写手 → 海外中文媒体 / 跨境电商文案
- **第一周做什么**:
  1. 注册 Upwork / Fiverr 账号
  2. 找 3 个你这个技能"成交价最低"的类目(说明竞争少 / 没人会)
  3. 写一份英文 profile,挂出 1-2 个 demo
  4. 主动投 5 个标的

#### A1.2 把英文技能搬到国内市场

- **前提**: 你会英文,看到国外有但国内没有的产品/服务/内容形式。
- **例子**:
  - 海外 SaaS 工具 → 汉化版 / 代理权 / 教程
  - 英文播客的中文摘要
  - 海外独立游戏的本地化运营
- **第一周做什么**:
  1. 列 10 个你常用的英文工具,看哪些国内没有
  2. 选 1 个做"中文体验优化"(教程/汉化/代理)
  3. 在小红书 / 知乎发 1 篇测评
  4. 看哪条评论反馈最强,继续做

#### A1.3 把线下技能搬到线上

- **前提**: 你有某个领域的线下经验(教培/咨询/手艺),想数字化。
- **例子**:
  - 线下家教 → 录播课 / 1v1 视频咨询
  - 线下瑜伽教练 → 跟练视频 + 会员社群
  - 线下烘焙 → 食谱订阅 + 食材盒
- **第一周做什么**:
  1. 列出你这技能"客户最常问的 5 个问题"
  2. 把 5 个问题录成 5 段 5 分钟视频
  3. 挂到小红书 / B站 / YouTube
  4. 看哪个问题播放最高,做付费版

## A2. 跨地区价差(原 #1 + #213 "跨市场跨界找资源")

### 适用场景

同一商品/服务,不同地区价格差异大,做"搬运工"。

### 三个具体操作

#### A2.1 实体商品跨地区

- **例子**:
  - 一线城市的进口零食/化妆品 → 拼多多/淘宝卖给三四线
  - 东南亚特色商品(泰国乳胶枕/越南咖啡) → 国内淘宝/小红书
  - 国内工厂尾货 → 拼多多低价清仓
- **第一周做什么**:
  1. 在拼多多 / 1688 找一个类目,看价格区间
  2. 在小红书 / 抖音搜同一商品,看买家愿付价格
  3. 算价差空间,选 1 个 SKU 做小规模测
  4. 用"代发"模式,自己不囤货

#### A2.2 服务跨地区

- **例子**:
  - 国内设计师给海外华人做 logo / 品牌设计
  - 国内程序员给海外小团队做外包
  - 国内文案给跨境电商写英文 listing
- **第一周做什么**:
  1. 在 Twitter / Reddit 找 3 个目标客户群
  2. 发"自我介绍 + 服务"贴文
  3. 主动 DM 10 个人
  4. 接 1 单就足够验证

#### A2.3 信息跨地区

- **例子**:
  - 海外学术资料的中文摘要
  - 国内政策的英文解读(给海外华人)
  - 国内行业报告的海外搬运
- **第一周做什么**:
  1. 选 1 个细分领域(你已经熟悉的)
  2. 找 10 个一手海外来源
  3. 做"中文版每周摘要"
  4. 发到小红书 + 邮件订阅

## A3. AI 杠杆(原 #5 "精通使用 ChatGPT 等 AI 工具")

### 适用场景

你有一个"本来靠时间换钱"的业务,想用 AI 把边际时间压到接近 0。

### 三个具体操作

#### A3.1 内容生产 AI 化

- **例子**:
  - 公众号文章 → AI 写初稿,人工润色
  - 小红书爆款图 → AI 生成 + 微调
  - YouTube 视频脚本 → AI 写脚本 + 人工改
- **第一周做什么**:
  1. 选 1 个内容渠道(你已经发的)
  2. 用 AI 生成 10 篇内容,人工选最好的 3 篇发布
  3. 对比这 3 篇 vs 你以前手写的流量
  4. 如果 AI 版 ≥ 手写版 70% 流量,把工作流改为"AI 主 + 人工润色"

#### A3.2 服务交付 AI 化

- **例子**:
  - 1v1 咨询 → 录一次深度对话 + AI 客服 + 偶尔人工
  - 设计服务 → AI 出 N 个版本,人工选 + 微调
  - 翻译服务 → AI 翻译 + 人工校对
- **第一周做什么**:
  1. 列出你这服务"哪些环节最耗时"
  2. 选 1 个环节试用 AI 替代
  3. 对比"AI 版"和"人工版"的质量差异
  4. 如果 ≥ 80% 质量,把这一环节外包给 AI

#### A3.3 信息检索 AI 化

- **例子**:
  - 行业研究 → 让 AI 抓 + 整理 + 摘要
  - 客户背景调查 → AI 搜 + 整理成报告
  - 竞品监控 → AI 自动抓 + 报警
- **第一周做什么**:
  1. 找 1 个你每周都要做的"信息搜集任务"
  2. 让 AI 跑一遍,看结果质量
  3. 如果 ≥ 80% 准确,把任务改成"AI 主 + 人工复核"

## 三类模式的边界

| 模式 | 适用前提 | 不适用场景 |
|---|---|---|
| A1 技能可复用 | 你已经会一项技能 | 完全从 0 开始的领域 |
| A2 跨地区价差 | 你能识别真实价差 | 红海类目(价差已被卷没) |
| A3 AI 杠杆 | 你的业务有可自动化环节 | 高度依赖人际关系/信任的业务 |

**第一周做什么** 永远要可执行,不是"调研一下""想想清楚"——这是原作者 #135 明确反对的。
