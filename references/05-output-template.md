# 05 — 输出模板(终端友好)

> V2 输出格式严格按本骨架,不堆 markdown,不堆 emoji。
> 终端直接可读,适合 agent 在对话里直接发给用户。

---

## 标准输出骨架

```
=== 机会发现雷达 v2 ===
假设画像: <技能背景> / <本金量级> / <时间预算> / <风险承受> / <地理>
框架符合度: 4 原则全过 / N 条不通过(<列出哪几条>)
合规状态: clear / 边缘需自评合规

【候选 1】<一句话名字>
  模式     : A1 技能可复用 / A2 跨地区价差 / A3 AI 杠杆
  本金门槛 : <具体数字>
  试错成本 : 占总资产 <X>%(<最坏情况数字>)
  预期收益 : <年化/月化区间,基于同类案例>
  打分     : D1=X D2=X D3=X D4=X D5=X D6=X → <总分>/60
  第一周   : <5 步以内可执行清单>
  反模式   : <7 条全勾,哪条 FAIL 写哪条>
  合规     : clear / 边缘(需自评)
  黑天鹅   : <1-2 个最大风险>
  IP 化    : <如何变个人品牌>
  提醒     : <一句金句,总结这个候选>

【候选 2】...
【候选 3】...
【候选 4】...
【候选 5】...

=== 排序 ===
推荐顺序: <#1 分数> > <#3 分数> > ...
投入产出比之王: <候选 X>
最易复制: <候选 Y>
最长期主义: <候选 Z>

=== 通用提醒 ===
- 同时跑 2-3 个候选,不 all-in
- 每个项目预算上限 = 总资产 1%
- 连续 2 个月无气色就砍掉
- 本 skill 不做股票/币种/具体项目买卖建议

=== 出处 ===
方法论来自某匿名知乎高赞作者的轻资产套利框架(2022-2025),引用金句标 #编号。
```

## 边界场景的输出

### 用户问"该不该买 XX 股票"

```
这不是本 skill 的范围。本 skill 只评估副业/兼职/转型方向的"机会框架",
不做具体股票/币种买卖建议。
建议:这类问题咨询持牌投资顾问。
```

### 用户问"XX 灰色生意能做吗"

```
明确违法 → 不做,本 skill 不展开
边缘灰色(如:跨境电商/海外注册) → 可以讨论,但所有候选必须标"需自评合规"
```

### 用户没给任何上下文

```
=== 机会发现雷达 v2 ===
假设画像: 技能中性 / < 10 万本金 / 副业形式 / 0 风险承受 / 国内(默认值,如果你不是请告诉我)
信号源: 待你提供(我才能套框架)

我需要你告诉我:
1. 你现在有什么技能(写作/设计/编程/翻译/...)
2. 你有多少时间(每天 2h / 周末 / 全职)
3. 你能接受的最大亏损是多少
4. 你在哪个地区(国内 / 海外 / 计划出海)
```

## 不要做的事

- ❌ 不要把金句用 markdown 加粗 `**...**` 包裹,直接纯文本
- ❌ 不要用 emoji(除了 `✓` 和 `✗` 用于判定)
- ❌ 不要用表格语法 `|...|`,终端里不好看,直接用对齐空格
- ❌ 不要在每个候选前加编号 `1.` `2.`,用 `【候选 N】` 形式
- ❌ 不要复述 SKILL.md 的内容,直接给打分和判断
