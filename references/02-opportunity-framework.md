# 02 — 6 维打分卡(候选商机评估框架)

> 用 6 维度对每个候选商机打分,**总分 ≥ 42/60 才推荐**。
> 这是 V2 的核心评估工具,任何用户提到的机会都先套这个打分卡。

---

## 6 维度定义

### D1 试错成本(0-10)

| 分 | 标准 |
|---|---|
| 0 | 需要贷款梭哈(总资产 50%+) |
| 3 | 5-10 万 启动,输了心疼但不死 |
| 5 | 1-5 万 启动,输了能挺过来 |
| 8 | 5000 元以下 启动,输了不痛 |
| 10 | ≤ 1000 元 / 0 现金启动(纯时间投入) |

### D2 经验可回收(0-10)

| 分 | 标准 |
|---|---|
| 0 | 失败 = 钱打水漂,什么也没学到 |
| 3 | 失败能学到一点皮毛 |
| 5 | 失败能涨可迁移技能(如:写代码/做视频/谈客户) |
| 8 | 失败涨经验 + 人脉 + 复利能力 |
| 10 | 失败本身就是"下次能成功"的必要训练 |

### D3 中间商 / 撮合度(0-10)

| 分 | 标准 |
|---|---|
| 0 | 重资产开店/开厂,自己生产 |
| 3 | 部分轻,部分要囤货 |
| 5 | 平台型/服务型,自己不做产品 |
| 8 | 纯撮合,一端接需求一端接供给 |
| 10 | 纯信息中介,零库存零边际成本 |

### D4 跨市场/跨境加成(0-10)

| 分 | 标准 |
|---|---|
| 0 | 完全国内封闭,无法跨境 |
| 3 | 国内多地区套利(如:一线内容 → 三四线) |
| 5 | 中↔英 单向套利 |
| 8 | 双向套利(英文 + 中文都能用) |
| 10 | 完全靠跨境信息差,国内做不了 |

### D5 新兴市场(0-10)

| 分 | 标准 |
|---|---|
| 0 | 房地产 / 传统餐饮 / 线下零售 |
| 3 | 短视频 / 直播带货(已是红海) |
| 5 | 公众号 / 知乎 / 小红书(增长见顶) |
| 8 | AI 应用 / 独立开发者 / 跨境电商 |
| 10 | AI 全链条 / 加密 / 跨境服务(但仅限合法范围) |

### D6 可 IP 化 + 全球化(0-10)

| 分 | 标准 |
|---|---|
| 0 | 一次性卖货,卖完拉倒 |
| 3 | 能做短视频,但无复利 |
| 5 | 能发内容积累粉丝 |
| 8 | 个人品牌 + 粉丝主动找上门 |
| 10 | 个人品牌 + 资产全球化(海外账号/收款/品牌) |

---

## 总分判定

- **≥ 50**: 强烈推荐,值得 2 周内启动
- **42-49**: 推荐,可以小步试错
- **35-41**: 谨慎,需要补强某些维度
- **< 35**: 不推荐,告诉用户"为什么不够格"

---

## 6 维度之间的权衡

不是 6 项都满分才好。**三个最关键的维度是 D1 / D2 / D4**:

- **D1 + D2** = 风险底盘(成本 + 学习)
- **D4** = 信息差来源(核心壁垒)

如果 D1 ≥ 8 但 D4 ≤ 3 → 是个好副业,但没有壁垒,容易被卷
如果 D4 ≥ 8 但 D1 ≤ 3 → 是个好机会,但门槛太高,先攒本金

## 评估示例:3 类典型候选

### 候选 A:AI 头像定制(SaaS)

- D1: 8(服务器 + 设计时间 ≤ 1000 元)
- D2: 7(失败学到 prompt engineering + 客户沟通)
- D3: 7(纯服务,不囤货)
- D4: 5(主要做国内市场,英文版可拓展)
- D5: 8(AI 应用,新兴)
- D6: 7(可发小红书案例积累)
- **总分: 42** → 推荐,可以试

### 候选 B:开一家奶茶店

- D1: 1(20-50 万,输了伤筋动骨)
- D2: 3(失败学到一点选址经验)
- D3: 1(重资产开店)
- D4: 0(纯本地,无法跨境)
- D5: 2(传统餐饮,红海)
- D6: 1(无 IP 价值)
- **总分: 8** → **PASS**(违反 P1 轻资产 + D1 / D3 / D5 / D6 全面不及格)

### 候选 C:把国内精品独立游戏搬到 Steam 海外版

- D1: 7(签约费 + 翻译 + 上架 ≤ 1 万)
- D2: 8(失败学到海外发行流程 + 谈判经验)
- D3: 9(纯撮合,游戏厂商 + 海外玩家)
- D4: 10(完全跨境,国内做不了)
- D5: 7(独立游戏还在增长)
- D6: 8(可做"独立游戏海外发行"账号)
- **总分: 49** → 强烈推荐

---

## 警示信号

如果候选在 6 维里有 3 项以上 ≤ 3 分,**即使总分勉强过 42**,也要警惕——大概率是"框架里看着不错,实际很难落地"。

直接告诉用户:"框架得分 45,但 D1/D3/D5 三项都偏低,实际执行时会撞墙,建议先想清楚为什么这几项分低,而不是直接开搞。"
